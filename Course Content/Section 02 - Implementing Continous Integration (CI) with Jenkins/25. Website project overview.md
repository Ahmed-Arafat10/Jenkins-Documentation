## 25. Website Project Overview

### 🔹 Starting Point: GitHub Repository

* After registering with **GitHub**, you land on your **GitHub dashboard** (all repositories/projects).
* In this section, we will work on an **existing project** (common real-world scenario).
* The repository is provided in the course **resources folder**.

---

### 🔹 Forking the Project

* Since everyone needs their own copy, we will **fork** the repository.

* Steps:

    1. Log into GitHub (check profile icon on the top right).
    2. Click on **Fork** (top-right).
    3. Choose repository name (default recommended).
    4. Leave **“Copy only the main branch”** checked.
    5. Click **Create Fork**.

* Result → You now have a **personal copy** of the repository.

* It will display *“forked from …”* at the top → confirmation it’s your copy.

---

### 🔹 Accessing and Working With the Code

There are two main approaches:

1. **Cloning locally with Git**

    * Clone via SSH/HTTPS.
    * Open project in **Visual Studio Code (VS Code)** or another IDE.

2. **Using GitHub Codespaces (in-browser IDE)**

    * Navigate to **Code → Codespaces → Create codespace on main**.
    * This launches a **web-based VS Code environment**.
    * Benefits: No local setup required.

---

### 🔹 Setting Up Codespaces

Inside Codespaces (web VS Code):

* **UI Layout**:

    * Left: **Project files** (e.g., `README.md`, `package.json`).
    * Center: **Editor area** (open files, multiple tabs).
    * Bottom: **Terminal** (Linux shell).

* **Customization**:

    * Switch to **Dark theme** (less eye strain).
    * Enable **Whitespace rendering** → Settings → search “Whitespace” → set to “All”.

---

### 🔹 Project Technology

* The project is a **Node.js application** (JavaScript).
* Similar setup concepts apply to **Java, Python, or other languages**.
* **Dependencies** (third-party libraries/frameworks) are defined in `package.json`.

---

### 🔹 Running the Project

1. **Install dependencies**:

   ```bash
   npm install
   ```

    * Uses **npm (Node Package Manager)**.
    * Installs everything listed under `"dependencies"` in `package.json`.
    * Warnings are common → safe to ignore unless explicit errors appear.

2. **Start the application**:

   ```bash
   npm start
   ```

    * Launches a **development server**.
    * Opens the app automatically in a browser at:

      ```
      http://localhost:3000
      ```
    * App → simple website with rotating Jenkins logo + text.

3. **Stop the app**:

    * In terminal, press **CTRL + C**.

---

### 🔹 Why Learn These Steps?

* As developers: we must **understand how to run the project manually**.
* As DevOps engineers: we later **automate these steps** using Jenkins.
* Rule → **You cannot automate what you don’t understand.**

---

### 🔹 Using the Terminal

* Codespaces terminal = **Linux shell**.
* Can run regular commands: `mkdir`, `touch`, `sleep`, etc.
* Useful for testing, debugging, and executing build/start commands.
