---

# Key Concepts

## AWS S3 `cp` Command

The AWS CLI provides the `cp` command (copy), which is used to:

* Upload files to S3
* Download files from S3
* Copy between S3 locations

Basic syntax:

```bash
aws s3 cp <source> <destination>
```

Example:

```bash
aws s3 cp index.html s3://my-bucket/index.html
```

---

## S3 Path Format

When working with S3 in CLI:

```bash
s3://<bucket-name>/<file>
```

Example:

```bash
s3://learn-jenkins-20240518-1820/index.html
```

---

## Dynamic File Creation in Pipeline

Instead of relying on existing files, the pipeline creates a file using:

```bash
echo "Hello S3" > index.html
```

This ensures:

* The pipeline is self-contained
* No dependency on external files

# Jenkinsfile Snapshot (Initial Upload Version)

```groovy
stage('AWS') {
    agent {
        docker {
            image 'amazon/aws-cli'
            args "--entrypoint=''"
        }
    }

    steps {
        withCredentials([
            usernamePassword(
                credentialsId: 'my-aws',
                usernameVariable: 'AWS_ACCESS_KEY_ID',
                passwordVariable: 'AWS_SECRET_ACCESS_KEY'
            )
        ]) {
            sh '''
            aws --version

            echo "Hello S3" > index.html

            aws s3 cp index.html s3://learn-jenkins-20240518-1820/index.html
            '''
        }
    }
}
```

# Improvement — Remove Hardcoded Bucket Name

## Problem

Hardcoding:

```bash
s3://learn-jenkins-20240518-1820/index.html
```

Issues:

* Hard to maintain
* Easy to forget changing
* Not reusable

---

## Solution — Use Environment Variables

Define a variable inside the pipeline:

```groovy
environment {
    AWS_S3_BUCKET = 'learn-jenkins-20240518-1820'
}
```

---

## Updated Jenkinsfile Snapshot (Improved Version)

```groovy
stage('AWS') {
    environment {
        AWS_S3_BUCKET = 'learn-jenkins-20240518-1820'
    }

    agent {
        docker {
            image 'amazon/aws-cli'
            args "--entrypoint=''"
        }
    }

    steps {
        withCredentials([
            usernamePassword(
                credentialsId: 'my-aws',
                usernameVariable: 'AWS_ACCESS_KEY_ID',
                passwordVariable: 'AWS_SECRET_ACCESS_KEY'
            )
        ]) {
            sh '''
            echo "Hello S3" > index.html

            aws s3 cp index.html s3://$AWS_S3_BUCKET/index.html
            '''
        }
    }
}
```

---

## How This Works

Inside the shell:

```bash
$AWS_S3_BUCKET
```

is replaced with:

```bash
learn-jenkins-20240518-1820
```

Final executed command:

```bash
aws s3 cp index.html s3://learn-jenkins-20240518-1820/index.html
```

---

## Logs Confirmation

Logs show:

* Original command with variable
* Expanded command with actual value

This confirms:

* Variable substitution works correctly

# Practical Notes / Best Practices

## Avoid Hardcoding Values

Always replace:

* Bucket names
* Regions
* Paths

with variables.

---

## Keep Pipelines Reusable

Using variables allows:

* Easy reuse across environments
* Easy updates
* Cleaner code

---

## Use CLI Instead of Manual Upload

Automation benefits:

* Faster deployments
* No human error
* Repeatable builds

---

## Understand S3 Object Structure

Uploading:

```bash
index.html
```

creates:

* An **object** in S3
* With key:

```text
index.html
```

# Common Mistakes or Warnings

## Wrong Bucket Name

If bucket name is incorrect:

```bash
aws s3 cp ...
```

→ Upload fails

---

## Missing Credentials

If `withCredentials` is missing:

```bash
Unable to locate credentials
```

---

## Forgetting `s3://` Prefix

Incorrect:

```bash
aws s3 cp index.html my-bucket
```

Correct:

```bash
aws s3 cp index.html s3://my-bucket/index.html
```

---

## Overwriting Files

Uploading the same file again:

* Replaces existing object
* No warning
