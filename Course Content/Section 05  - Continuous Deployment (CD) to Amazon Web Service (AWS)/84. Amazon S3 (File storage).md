# Lecture 84 — Amazon S3 (File Storage)

## Overview

This lecture introduces **Amazon S3 (Simple Storage Service)**, the first AWS service used in the course. Amazon S3 is a highly scalable **object storage service** used to store and retrieve files.

In this course, S3 will be used to **host and serve the static website** built earlier in the Jenkins pipeline. Since the website does not require a backend server or database, it can be deployed directly to S3 and served over HTTP.

The lecture demonstrates how to:

* Access the **Amazon S3 service from the AWS Management Console**
* Understand the **concept of buckets and objects**
* Create an **S3 bucket**
* Navigate the **S3 interface**

Later lectures will automate this deployment using **Jenkins and the AWS CLI**, instead of uploading files manually.

---

## Key Concepts

### Amazon S3 (Simple Storage Service)

**Amazon S3** is a cloud storage service designed to store and retrieve any amount of data.

Key characteristics:

* Highly scalable
* Highly durable
* Globally accessible
* Designed for storing files as **objects**

S3 is commonly used for:

* Static website hosting
* Backup storage
* Media storage
* Data archives
* Application assets

A useful comparison:

> Amazon S3 can be thought of like **Dropbox**, but designed for **developers, automation, and infrastructure systems**.

*(Fun fact mentioned in the lecture: Dropbox previously used Amazon S3 internally to store files.)*

---

### Buckets

In Amazon S3, files are stored inside **buckets**.

A **bucket** is essentially a container for objects.

You can think of a bucket as:

> A **large folder** that stores files in the cloud.

Important characteristics of buckets:

* Bucket names must be **globally unique**
* Buckets belong to a **specific AWS region**
* Buckets can store **unlimited objects**

Example bucket name:

```
learn-jenkins-20240518-1820
```

Using **timestamps or unique identifiers** helps avoid name conflicts.

---

### Objects

Files stored inside a bucket are called **objects**.

An **object** consists of:

* The file itself
* Metadata about the file
* A unique key (the object's path within the bucket)

Examples of S3 objects:

```
index.html
styles.css
app.js
logo.png
```

These files will represent the **static website assets**.

## Detailed Explanation

### Accessing Amazon S3

To start working with S3:

1. Open the **AWS Management Console**.
2. Locate the **search bar at the top of the interface**.
3. Type:

```
S3
```

4. Select the first result called:

```
S3
```

This will open the **Amazon S3 service page**.

The first time you open S3, you will see an **introductory page** containing general information about the service.

The interface may look slightly different from what is shown in the course because:

> AWS frequently updates the user interface.

However, the **core concepts and workflow remain the same**.

---

## Step-by-Step Process — Creating an S3 Bucket

### Step 1 — Open the S3 Service

1. Go to the **AWS Management Console**.
2. Use the search bar.
3. Search for:

```
S3
```

4. Click the **S3 service**.

---

### Step 2 — Create a Bucket

Click:

```
Create bucket
```

You will see a bucket configuration page.

---

### Step 3 — Choose a Bucket Name

One of the most important settings is the **bucket name**.

Requirements:

* Must be **globally unique**
* Must follow AWS naming rules
* Cannot already exist anywhere in S3

Example naming strategy used in the lecture:

```
learn-jenkins-20240518-1820
```

The name includes:

* Project identifier
* Date
* Time

This helps ensure uniqueness.

If the bucket name already exists, AWS will show an **error message**.

---

### Step 4 — Keep Default Settings

For this course setup:

* Most configuration settings can remain **default**
* No additional configuration is required at this stage

Scroll to the bottom of the configuration page.

---

### Step 5 — Create the Bucket

Click:

```
Create bucket
```

Once completed, the bucket will appear in the **S3 bucket list**.

---

## Navigating the S3 Interface

After the bucket is created, you will see the **S3 dashboard** listing all buckets.

In this example:

* Only **one bucket exists**

From this page you can:

* Create new buckets
* Open existing buckets
* Manage stored files

To view the bucket contents:

1. Click the **bucket name**.
2. The bucket page will open.

Since the bucket is newly created, you will see:

```
No objects
```

This means the bucket currently contains **no files**.

## Code Examples

At this stage, the bucket is created **manually via the AWS console**.

Later lectures will automate file uploads using **AWS CLI commands** inside a Jenkins pipeline.

Example concept of uploading files to S3:

```bash
aws s3 sync build/ s3://learn-jenkins-20240518-1820
```

This command would:

* Upload all files from the `build/` directory
* Synchronize them with the S3 bucket

Example Jenkins pipeline stage:

```groovy
stage('Deploy to S3') {
    steps {
        sh 'aws s3 sync build/ s3://learn-jenkins-20240518-1820'
    }
}
```

## Practical Notes / Best Practices

* Always choose **unique bucket names**.
* Using **timestamps or UUIDs** helps prevent naming conflicts.
* Keep track of your **AWS region**, since buckets are region-specific.
* Use **automation tools** (like Jenkins pipelines) instead of manual uploads for production deployments.

Automation ensures:

* Repeatable deployments
* Reduced human error
* Faster delivery pipelines

---

## Common Mistakes or Warnings

### Bucket Name Already Exists

S3 bucket names are **globally unique across AWS**.

If you attempt to create a bucket with an existing name, AWS will display an error.

Solution:

* Add **timestamps or random identifiers** to the name.

---

### Relying on Manual Uploads

While the S3 console allows manual file uploads, this is **not suitable for CI/CD pipelines**.

In automated environments:

* Deployments should be performed via **CLI tools or automation scripts**.
