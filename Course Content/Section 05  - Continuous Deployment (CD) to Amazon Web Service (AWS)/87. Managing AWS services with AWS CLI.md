# Lecture 87 — Managing AWS Services with AWS CLI

## Overview

In this lecture, we move from simply **running the AWS CLI** to actually **using it to manage AWS services**.

The focus is on:

* Understanding how to use the **AWS CLI command reference**
* Learning how to interact with **specific AWS services (like S3)**
* Attempting to run a real command (`aws s3 ls`)
* Understanding why the command fails due to **missing authentication (credentials)**

This lecture introduces a critical concept:

> The AWS CLI must be **authenticated** before it can access any AWS resources.

---

# Key Concepts

## AWS CLI Command Structure

To interact with AWS services, commands follow this structure:

```bash
aws <service> <command>
```

Example:

```bash
aws s3 ls
```

Breakdown:

* `aws` → CLI tool
* `s3` → AWS service (Amazon S3)
* `ls` → command (list resources)

---

## AWS CLI Command Reference

The **AWS CLI documentation** provides a complete list of:

* Supported services
* Available commands
* Command syntax
* Parameters and options

Inside the documentation:

* First section → **Global options**
* Later section → **Available services**

Each service (like S3) has its own set of commands.

---

## S3 Commands via AWS CLI

When navigating to the **S3 section** in the AWS CLI documentation, you will find commands such as:

* `ls` → List buckets or objects
* `cp` → Copy files
* `sync` → Synchronize directories
* `rm` → Remove files

Example command:

```bash
aws s3 ls
```

This command is expected to:

* List all S3 buckets in your AWS account
* Equivalent to what you see in the AWS Console

## Step 4 — Pipeline Execution Result

After running the pipeline:

* The first command works:

```bash
aws --version
```

* The second command fails:

```bash
aws s3 ls
```

---

## Error Output

```bash
Unable to locate credentials. You can configure credentials by running "aws configure".
```

---

# Why This Error Happens

The AWS CLI does not know:

* Who you are
* Which AWS account to access

Important point:

> The Docker image used in Jenkins is generic — it has **no authentication configured**.

If AWS CLI allowed access without credentials:

* Anyone could access any AWS account
* This would be a **major security issue**

Therefore, AWS requires **authentication before executing any command that accesses resources**.

---

# Authentication Requirement

To use AWS CLI commands like:

```bash
aws s3 ls
```

You must provide:

* Access credentials
* Identity information

Typically, this includes:

* Access Key ID
* Secret Access Key

*(These act like a username and password for programmatic access.)*

---

# Why `aws configure` Is Not Used in Jenkins

The error suggests:

```bash
aws configure
```

However, this command is:

* **Interactive**
* Requires user input (keyboard prompts)

Jenkins pipelines:

* Are **non-interactive**
* Cannot accept manual input during execution

Therefore:

> `aws configure` is NOT suitable for CI/CD pipelines.

# Practical Notes / Best Practices

## Always Expect Authentication for Cloud Commands

Any AWS CLI command that:

* Reads data
* Writes data
* Modifies resources

Will require authentication.

---

## Understand the Security Model

AWS CLI is secure by design:

* No credentials → No access
* Prevents unauthorized access to cloud resources

---

## Use Documentation to Discover Commands

Instead of memorizing commands:

* Use the **AWS CLI command reference**
* Search for services and commands
* Understand syntax before usage

---

## Think in Terms of Services

Every AWS CLI command follows this logic:

```bash
aws <service> <action>
```

Examples:

* `aws s3 ls`
* `aws ec2 describe-instances`
* `aws iam list-users`

---

# Common Mistakes or Warnings

## Trying to Run AWS CLI Without Credentials

This is the most common beginner mistake.

Result:

```bash
Unable to locate credentials
```

Solution:

* Configure authentication (covered in upcoming lectures)

---

## Expecting CLI to Work Like the UI

The AWS Console:

* Automatically uses your logged-in session

The AWS CLI:

* Requires explicit credentials

These are **completely different authentication models**.

---

## Attempting Interactive Commands in Jenkins

Commands like:

```bash
aws configure
```

will not work in Jenkins because:

* Jenkins cannot provide interactive input
