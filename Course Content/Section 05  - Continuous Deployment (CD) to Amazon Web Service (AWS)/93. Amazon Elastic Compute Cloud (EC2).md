# Lecture 93 — Amazon Elastic Compute Cloud (EC2)

## Overview

This lecture introduces **Amazon EC2 (Elastic Compute Cloud)** — one of the most fundamental services in AWS.

EC2 allows you to:

* Launch **virtual machines (servers)** in the cloud
* Run applications, services, or scripts
* Control computing resources on demand

Even though EC2 is **not directly used in this course**, understanding it is essential because:

> Many AWS services are built on top of EC2.

---

# Key Concepts

## What is EC2?

**Amazon EC2** is a service that provides:

> Virtual servers (computers) that run in AWS data centers.

Think of EC2 as:

* A **remote Linux or Windows machine**
* That you can start, stop, and configure anytime

---

## EC2 Instance

An **instance** is:

* A running virtual machine in EC2

Each instance has:

* CPU
* Memory
* Storage
* Network configuration

---

## Amazon Machine Image (AMI)

An **AMI** is a template used to create an instance.

It includes:

* Operating system (e.g., Ubuntu, Amazon Linux)
* Pre-installed software

Example:

```text
Ubuntu Server 24.04
```

---

## Instance Type

Defines how powerful the machine is.

Example:

```text
t2.micro
```

* Free-tier eligible
* Small CPU and memory
* Ideal for learning

---

## Free Tier

AWS provides:

* ~750 hours/month of **t2.micro / t3.micro**

Important:

* Stay within free-tier resources to avoid charges

# Understanding EC2 Behavior

## Idle State

After launch:

* Instance is running
* But doing nothing

You must:

* Run commands
* Start services
* Deploy applications

---

## No GUI

EC2 instances:

* Typically **headless (no graphical interface)**
* Controlled via terminal

---

# Managing EC2 Lifecycle

## Stop Instance

Equivalent to:

> Shutting down your computer

Effects:

* Instance stops
* You are NOT charged for compute
* Storage (EBS) still costs money

---

## Start Instance

* Restart stopped instance
* Same configuration resumes

---

## Terminate Instance

Equivalent to:

> Deleting the machine permanently

Effects:

* Instance is destroyed
* Storage is deleted
* Cannot be recovered

---

# Important: Elastic IP

An **Elastic IP** is:

* A public IP address assigned to your instance

Important:

* May incur cost if unused
* Should be released when no longer needed

---

# Step-by-Step — Stop & Terminate

## Stop Instance

```text
Instance State → Stop Instance
```

---

## Terminate Instance

```text
Instance State → Terminate Instance
```

✔️ Removes instance completely
✔️ Prevents future charges

---

## Check Elastic IPs

Navigate:

```text
Network & Security → Elastic IPs
```

Ensure:

* No unused IPs remain

# Summary

Amazon EC2 is a core AWS service that provides **virtual servers in the cloud**.

Key takeaways:

* EC2 instances are **virtual machines**
* AMIs define the operating system
* Instance types define performance
* You can connect via **browser terminal**
* Instances must be:

    * **Stopped** (pause)
    * **Terminated** (delete) to avoid costs
* EC2 is foundational for many AWS services

Even though EC2 is not directly used in this course:

> Understanding EC2 helps you understand how cloud infrastructure works under the hood.
