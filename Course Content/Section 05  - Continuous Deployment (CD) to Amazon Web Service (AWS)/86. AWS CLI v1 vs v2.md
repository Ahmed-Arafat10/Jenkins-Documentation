# Lecture 86 — AWS CLI v1 vs v2

## Overview

The AWS Command Line Interface currently exists in **two major versions**:

* **AWS CLI Version 1**
* **AWS CLI Version 2**

While both versions allow interaction with AWS services from the command line, **Version 2 is the modern and recommended version**.

This lecture explains the differences between the two versions and highlights an important issue: **AWS documentation links can sometimes point to the older Version 1 documentation**, which may cause confusion.

Throughout this course, **AWS CLI Version 2 will be used exclusively**.

---

# Key Concepts

## AWS CLI Version 1

**AWS CLI Version 1** is the original implementation of the AWS command line tool.

Characteristics:

* Much older implementation
* Still available for compatibility
* Often referenced in older documentation
* Not recommended for new environments

Although it still functions, most modern environments have migrated to **Version 2**.

---

## AWS CLI Version 2

**AWS CLI Version 2** is the current and recommended version of the AWS CLI.

It provides:

* Improved usability
* Better packaging and distribution
* Additional built-in features
* Improved security and authentication support

Most new AWS tooling and documentation now assumes **Version 2**.

---

# Detailed Explanation

## Documentation Confusion

One common issue when working with AWS CLI is that **documentation links may lead to Version 1 documentation even if they appear to reference the latest version**.

For example, documentation URLs may include:

```
/cli/latest/
```

This might appear to mean **latest version**, but in many cases it actually refers to:

> The latest documentation for **AWS CLI Version 1**, not Version 2.

Because of this, developers must verify that they are reading the correct documentation.

---

## Identifying Version 2 Documentation

To ensure you are using the correct documentation:

1. Verify the page explicitly references **AWS CLI Version 2**.
2. Use links that specifically mention **v2**.
3. Follow links provided in reliable sources (such as course material).

The correct documentation should clearly indicate:

```
AWS CLI Version 2
```

---

# Step-by-Step Process — Verifying CLI Version

It is good practice to verify the installed AWS CLI version in your environment.

Command:

```bash
aws --version
```

Example output:

```
aws-cli/2.15.53 Python/3.x Linux/amd64
```

Key detail:

* The version number begins with **2.x**

This confirms that **AWS CLI Version 2** is installed.

---

# Code Examples

## Checking AWS CLI Version in Jenkins

Inside a Jenkins pipeline stage, you can confirm the CLI version:

```groovy
stage('Check AWS CLI') {
    agent {
        docker {
            image 'amazon/aws-cli'
            args "--entrypoint=''"
        }
    }

    steps {
        sh 'aws --version'
    }
}
```

This ensures that the pipeline environment is using **AWS CLI Version 2**.

# Practical Notes / Best Practices

### Always Use AWS CLI v2

For new projects and CI/CD pipelines:

* Prefer **AWS CLI Version 2**
* Avoid relying on older CLI versions

Version 2 receives:

* Active maintenance
* Feature updates
* Security improvements

---

### Verify Documentation Version

When reading AWS documentation:

Always check:

* Whether the page refers to **v1 or v2**
* Whether the commands match your CLI version

Incorrect documentation may lead to:

* Invalid commands
* Unexpected behavior
* Misconfigured automation scripts

---

### Confirm Version in Pipelines

Including version checks in pipelines can help with debugging:

```bash
aws --version
```

This makes it easier to identify when:

* A Docker image changed
* The CLI version changed
* Commands stopped working after updates

---

# Common Mistakes or Warnings

## Using Version 1 Documentation by Accident

A common mistake is following documentation intended for **AWS CLI Version 1** while using **Version 2**.

This can lead to:

* Incorrect command syntax
* Missing features
* Misconfigured commands

Always confirm that documentation explicitly refers to **AWS CLI v2**.

---

## Assuming "Latest" Means Version 2

The term **latest** in AWS documentation URLs may refer to the **latest version of CLI v1 documentation**, not CLI v2.

Developers must verify the documentation version explicitly.

---
