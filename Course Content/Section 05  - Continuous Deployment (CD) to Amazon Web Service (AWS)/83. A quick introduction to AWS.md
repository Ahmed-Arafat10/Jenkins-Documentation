# Lecture 83 — A Quick Introduction to AWS

## Overview

This lecture introduces **Amazon Web Services (AWS)**, one of the most widely used cloud computing platforms.

AWS provides a large ecosystem of **cloud services and infrastructure** that organizations use to build, deploy, and scale applications. Instead of purchasing and maintaining physical hardware, developers and companies can **rent computing resources from AWS on demand**.

In this course section, AWS will be used as the **deployment platform for the website project** built earlier in the Jenkins pipeline.

---

## Key Concepts

### Amazon Web Services (AWS)

**Amazon Web Services (AWS)** is a cloud computing platform offering **over 200 cloud services** hosted in data centers around the world.

These services allow organizations to build and operate infrastructure without owning physical hardware.

Common AWS service categories include:

* **Compute services**

    * Virtual servers
    * Container infrastructure
* **Storage services**

    * Object storage
    * File storage
* **Databases**

    * Managed relational and NoSQL databases
* **Networking**

    * Load balancing
    * Content delivery networks
* **Security**

    * Identity and access management

AWS operates **global data centers** that provide high availability, scalability, and reliability.

---

### Pay-As-You-Go Pricing Model

AWS follows a **pay-as-you-go pricing model**.

Instead of buying hardware, organizations pay only for the **resources they actually consume**.

This model allows companies to:

* Avoid large upfront infrastructure investments
* Scale infrastructure dynamically
* Reduce operational overhead

Typical resources billed in AWS include:

* Compute time
* Storage usage
* Data transfer
* Database usage

---

### Cloud Infrastructure vs Traditional Infrastructure

Traditional infrastructure requires organizations to:

* Purchase servers
* Install hardware in data centers
* Manage cooling, power, and networking
* Maintain hardware

With AWS:

* Infrastructure is **provisioned instantly**
* Resources are **managed by AWS**
* Infrastructure can be **scaled automatically**

This dramatically simplifies the process of deploying applications.

## Detailed Explanation

### Growth of AWS

AWS began in the **early 2000s** and has since become one of the largest cloud platforms in the world.

Over time, many organizations have migrated their infrastructure to cloud environments.

Today, many companies rely heavily on cloud services for:

* Hosting applications
* Data processing
* Data storage
* Machine learning
* Global service delivery

In modern DevOps environments, cloud infrastructure is often tightly integrated with **CI/CD pipelines**.

---

### Real-World Example: The New York Times

An early example demonstrating the power of cloud computing involved **The New York Times**.

The organization wanted to **digitize over one million archived articles**.

Using AWS infrastructure:

* A software architect converted **scanned article images into PDF files**
* The entire process was completed in **less than 24 hours**
* The cost was **significantly lower than traditional infrastructure solutions**

This example demonstrated how **cloud computing could process large workloads quickly and economically**.

## Creating an AWS Account

To use AWS services, you must first create an **AWS account**.

### Step-by-Step Process

1. Go to the AWS registration page.

2. Enter your **personal details**.

3. Choose an **account type**.

Recommended choice for learning:

* **Personal account**

This is ideal for:

* Learning
* Experimentation
* Small projects

4. Provide a **credit card or debit card**.

Even though AWS offers a **Free Tier**, payment details are still required in case usage exceeds the free limits.

*(The Free Tier provides limited usage of many services at no cost for new accounts.)*

5. Choose a **support plan**.

Recommended option for learners:

* **Basic Support Plan (Free)**

This is sufficient for individuals and experimentation.

Once the account setup and verification are complete, AWS can be used immediately.

---

## Accessing the AWS Management Console

After creating an AWS account, sign in to the **AWS Management Console**.

The **AWS Management Console** is the web interface used to manage AWS services.

If you successfully reach the console dashboard, it means:

* The account was created correctly
* Billing information was verified
* You can begin using AWS services

---

## Navigating the AWS Console

The **AWS Management Console** is the central interface for interacting with all AWS services.

From this dashboard you can:

* Browse available services
* Launch infrastructure
* Monitor resources
* Configure settings

### Service Navigation

There are two main ways to access AWS services:

1. **Services List**

AWS provides a full catalog of available services.

2. **Search Bar**

You can search directly for a service by name.

Example services you might search for:

* S3
* EC2
* Lambda
* RDS

---

### AWS Regions

AWS infrastructure is distributed across **multiple global regions**.

A **region** represents a specific geographic location containing multiple AWS data centers.

Examples include:

* US East (North Virginia)
* Europe (Frankfurt)
* Asia Pacific (Tokyo)

In the course demonstration, the selected region is:

**US East (N. Virginia)**

Important notes:

* You may choose **any region you prefer**
* Some regions have **different pricing structures**
* Regions in the **United States are often cheaper**

However, the most important rule is:

> Always remember which region you selected because your resources will be created in that region.

This will be important later when deploying infrastructure.

## Security Recommendation: Multi-Factor Authentication

A critical security best practice for AWS accounts is enabling **Multi-Factor Authentication (MFA)**.

**Multi-Factor Authentication (MFA)** adds an additional security layer by requiring:

1. Your password
2. A temporary verification code from a device

Benefits of MFA:

* Protects the account even if the password is compromised
* Prevents unauthorized access
* Improves overall account security

It is highly recommended to enable MFA from the **Account Security Settings**.

## Practical Notes / Best Practices

* Choose a **Personal AWS account** for learning purposes.
* Always remember the **AWS region** where your resources are created.
* Use the **AWS search bar** to quickly find services.
* Enable **Multi-Factor Authentication (MFA)** immediately for security.
* Monitor usage to avoid **unexpected charges beyond the Free Tier limits**.

---

## Common Mistakes or Warnings

### Forgetting the Selected Region

Resources created in AWS are **region-specific**.

If you switch regions in the console, your resources may appear to be missing because they exist in another region.

Always verify the **selected region** in the console.

---

### Exceeding Free Tier Limits

AWS Free Tier offers limited usage.

If you exceed these limits, AWS may start charging your account.

Always monitor your:

* Compute usage
* Storage usage
* Data transfer
