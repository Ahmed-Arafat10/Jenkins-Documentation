## Lecture 75 – Using a Custom Docker Image in the Pipeline

### Objective

Now that we have built our custom Docker image (`my-playwright`), the next step is:

* Replace the base Playwright image
* Use our custom image in `deploy staging`
* Use it in `deploy prod`
* Remove repeated tool installation steps

This makes the pipeline:

* Faster
* Cleaner
* More efficient

---

# Step 1 – Replace the Base Image

Previously, we used:

```groovy
agent {
  docker {
    image 'mcr.microsoft.com/playwright:v1.40.0-foca
  }
}
```

Now we replace it with:

```groovy
agent {
  docker {
    image 'my-playwrigh
  }
}
```

Do this in:

* `Deploy Staging`
* `Deploy Prod`

---

# Step 2 – Remove Tool Installation

Previously inside stages we had:

```bash
npm install netlify-cli node-jq
```

Now:

* Netlify CLI is installed globally
* node-jq is installed globally
* No need to install them again

So remove:

```bash
npm install netlify-cli
npm install node-jq
```

---

# Step 3 – Update Tool Paths

Previously, when tools were locally installed:

```bash
node_modules/.bin/node-jq
```

Now, since installed globally:

```bash
node-jq
```

Similarly:

```bash
netlify deploy ...
netlify status
```

No need for local path references.

---

# Updated Deploy Staging Stage

```groovy
stage('Deploy Staging') {
  agent {
    docker {
      image 'my-playwright'
    }
  }

  steps {
    sh '''
      netlify deploy --dir=build --json > deploy.output

      CI_ENVIRONMENT_URL=$(node-jq -r '.deploy_url' deploy.output)

      echo "Running tests against $CI_ENVIRONMENT_URL"

      npx playwright test
    '''
  }

  post {
    always {
      publishHTML(target: [
        reportDir: 'playwright-report',
        reportFiles: 'index.html',
        reportName: 'Staging E2E'
      ])
    }
  }
}
```

---

# Updated Deploy Production Stage

```groovy
stage('Deploy Prod') {
  agent {
    docker {
      image 'my-playwright'
    }
  }

  steps {
    sh '''
      netlify deploy --dir=build --prod
      npx playwright test
    '''
  }

  post {
    always {
      publishHTML(target: [
        reportDir: 'playwright-report',
        reportFiles: 'index.html',
        reportName: 'Prod E2E'
      ])
    }
  }
}
```

---

# Performance Improvement

Observed improvements:

* Deploy Staging reduced significantly (example: ~1m 23s → ~34s)
* Deploy Prod also faster
* Entire pipeline execution time reduced

Why?

Because we removed:

* Repeated `npm install`
* Re-downloading dependencies
* Tool installation overhead

Now:

* Container starts
* Tools already available
* Execution begins immediately

---

# Why This Is a Better Design

### Before

Each pipeline run:

* Pulled base image
* Installed tools
* Executed commands

### After

Each pipeline run:

* Uses prebuilt image
* Tools already installed
* Direct execution

Cleaner separation:

* Image build responsibility → Dockerfile
* CI logic responsibility → Jenkinsfile

---

# Key Architectural Principle

Build environments once, reuse many times.

This reduces:

* Waste
* Execution time
* Network overhead
* CI instability

---

# Important Reminder

Since image is built locally:

* It must exist on the Jenkins machine
* If Jenkins agent changes, image must exist there too

(Handled later by pushing to registry in real-world setups.)
