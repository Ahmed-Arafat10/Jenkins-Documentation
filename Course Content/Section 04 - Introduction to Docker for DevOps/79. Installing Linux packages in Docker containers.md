# 🐳 Installing Linux Packages Inside Docker Agent (Jenkins)

The core idea is simple:

> You cannot use `npm` if Node is not installed.
> You must use the Linux package manager of the Docker image.

And every Linux distribution has its own package manager.

---

# 🏔️ Case 1: Alpine-Based Image (Python Alpine Example)

## Example Agent

```groovy
agent {
    docker {
        image 'python:3-alpine'
    }
}
```

Alpine uses:

```
apk
```

So installing `jq` becomes:

```groovy
sh 'apk add jq'
```

---

## 🚨 Problem: Permission Denied

You’ll get:

```
ERROR: Permission denied
```

Because Jenkins does NOT run containers as root by default.

---

## ⚠️ Solution (Not Always Recommended)

Run container as root:

```groovy
agent {
    docker {
        image 'python:3-alpine'
        args '-u root'
    }
}
```

Now this works:

```groovy
sh 'apk add jq'
```

---

## ⚠️ Important Warning

If:

* You use `reuseNode true`
* Workspace is shared
* You create files as root

Later Jenkins may fail deleting files due to permission issues.

This is why installing packages at runtime is **not ideal for production pipelines**.

---

# 🐧 Case 2: Ubuntu-Based Image

## Example Agent

```groovy
agent {
    docker {
        image 'ubuntu:noble'
    }
}
```

Ubuntu uses:

```
apt
```

---

## Step 1 — Update Package Index

Always required:

```groovy
sh 'apt update'
```

If you skip this, you’ll get:

```
Unable to locate package jq
```

---

## Step 2 — Install Package

```groovy
sh 'apt install jq'
```

---

## 🚨 Problem: Interactive Prompt

You’ll see:

```
Do you want to continue? [Y/n]
```

Jenkins cannot answer prompts.

---

## ✅ Solution: Use `-y`

```groovy
sh 'apt install -y jq'
```

Now installation is non-interactive.

---

# 🧠 Summary of Package Managers

| Distribution  | Package Manager | Install Example                   |
| ------------- | --------------- | --------------------------------- |
| Alpine        | apk             | `apk add jq`                      |
| Ubuntu/Debian | apt             | `apt update && apt install -y jq` |
| CentOS/RHEL   | yum/dnf         | `yum install -y jq`               |

---

# 🏆 Best Practice (Production-Grade Approach)

Instead of installing packages inside pipeline every build:

### 🔥 Build Custom Image Once

Example:

```dockerfile
FROM ubuntu:noble

RUN apt update && \
    apt install -y jq && \
    apt clean
```

Then in Jenkins:

```groovy
agent {
    docker {
        image 'my-custom-ubuntu'
    }
}
```

This is:

* Faster
* Cleaner
* More secure
* Avoids root problems
* Avoids repeated installation

---

# 🎯 When Should You Install Inside Pipeline?

Only for:

* Learning
* Temporary debugging
* Quick experimentation

Never for stable CI/CD production.
