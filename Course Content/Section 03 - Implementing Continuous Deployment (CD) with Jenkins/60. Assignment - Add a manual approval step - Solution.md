## Lecture 60 – Assignment: Add a manual approval step (Solution)

### Placement of the approval stage

The assignment requires adding a manual approval step:

* **after** `deploy staging`
* **before** `deploy production`

This ensures production deployment only happens after explicit approval.

---

### Creating the approval stage

A new stage named `approval` is added between the two deploy stages.

Basic structure:

```groovy
stage('Approval') {
  steps {
    // approval logic goes here
  }
}
```

---

### Adding the input (manual approval) step

Inside the `steps` block, Jenkins’ `input` step is used.

Using **Pipeline Syntax (Snippet Generator)**:

* select `input`
* set the message to:

```
Danger: approving this will result in a deployment to production.
Do you wish to deploy to production?
```

* expand **Advanced**
* customize the OK button text

Generated input step:

```groovy
input message: 'Danger: approving this will result in a deployment to production.\nDo you wish to deploy to production?',
      ok: 'Yes, I am sure'
```

---

### Approval stage without timeout (initial version)

```groovy
stage('Approval') {
  steps {
    input message: 'Danger: approving this will result in a deployment to production.\nDo you wish to deploy to production?',
          ok: 'Yes, I am sure'
  }
}
```

Pipeline behavior:

* `deploy staging` completes
* pipeline pauses at `Approval`
* Jenkins UI shows the approval dialog
* selecting **Yes, I am sure** allows the pipeline to continue to production

---

### Adding a timeout to the approval stage

Leaving the pipeline waiting indefinitely is not desirable.

A timeout of **15 minutes** is added around the input step.

Using Pipeline Syntax:

* select `timeout`
* duration: `15`
* unit: `minutes`

---

### Correct structure: timeout wrapping input

The `input` step must be **inside** the `timeout` block.

Final approval stage:

```groovy
stage('Approval') {
  steps {
    timeout(time: 15, unit: 'MINUTES') {
      input message: 'Danger: approving this will result in a deployment to production.\nDo you wish to deploy to production?',
            ok: 'Yes, I am sure'
    }
  }
}
```

Behavior:

* Jenkins waits up to 15 minutes for approval
* if approved → pipeline continues to production
* if no action is taken → pipeline is automatically aborted

---

### Full deploy flow with approval

```groovy
stage('Deploy Staging') {
  steps {
    echo 'Deploying to staging'
  }
}

stage('Approval') {
  steps {
    timeout(time: 15, unit: 'MINUTES') {
      input message: 'Danger: approving this will result in a deployment to production.\nDo you wish to deploy to production?',
            ok: 'Yes, I am sure'
    }
  }
}

stage('Deploy Production') {
  steps {
    echo 'Deploying to production'
  }
}
```

---

### Result of the assignment

* staging deployment runs automatically
* production deployment requires explicit approval
* approval is time-limited to avoid hanging pipelines
* pipeline behavior matches real-world CI/CD safety practices

---

### (Added by me – not mentioned in the lecture)

In production pipelines, approval messages are often enhanced with:

* the **staging URL**
* commit hash or version number
* change summary
  to help reviewers make an informed decision before approving deployment.
