## Lecture 63 – Post-deployment tests on staging

### Objective

The goal is to:

* run **end-to-end tests on staging**
* before the manual approval step
* using the dynamically generated staging URL

The new stage must be placed:

* after `deploy staging`
* before `approval`

---

## Creating the `staging e2e` stage

The production E2E stage is reused as a template.

New stage:

* copied from `prod e2e`
* renamed to `staging e2e`
* report name updated

Example structure:

```groovy
stage('Staging E2E') {
  agent {
    docker {
      image 'mcr.microsoft.com/playwright:v1.40.0-focal'
      args '-u root'
    }
  }
  environment {
    CI_ENVIRONMENT_URL = "FIXME"
  }
  steps {
    sh 'npx playwright test'
  }
  post {
    always {
      publishHTML(target: [
        reportDir: 'playwright-report',
        reportFiles: 'index.html',
        reportName: 'Staging E2E'
      ])
    }
  }
}
```

The only missing piece is:

* dynamically setting `CI_ENVIRONMENT_URL`

---

## Recap: extracting the staging URL

Inside `deploy staging`, the pipeline:

1. Runs Netlify deploy with `--json`
2. Redirects output to `deploy.output`
3. Uses `node-jq` to extract `.deploy_url`

Parsing command:

```bash
node_modules/.bin/node-jq -r '.deploy_url' deploy.output
```

Now this value must:

* be captured
* stored
* reused in `staging e2e`

---

## Setting a dynamic environment variable in `deploy staging`

Inside `deploy staging`, a `script` block is required.

Important:

* `script` must be inside `steps`
* not directly inside `stage`

Correct structure:

```groovy
stage('Deploy Staging') {
  steps {
    sh 'npm install netlify-cli node-jq'
    sh 'netlify deploy --dir=build --json > deploy.output'

    script {
      env.STAGING_URL = sh(
        script: "node_modules/.bin/node-jq -r '.deploy_url' deploy.output",
        returnStdout: true
      ).trim()
    }
  }
}
```

Key points:

* double quotes are used around the script string
* because the jq command already uses single quotes
* `.trim()` removes trailing newline

Common mistake:
Placing `script` outside `steps` causes:

```
Unknown stage section "script"
Steps in a stage must be in a steps block
```

---

## Passing the URL to Playwright in `staging e2e`

Playwright expects:

```
CI_ENVIRONMENT_URL
```

Inside `staging e2e`, set:

```groovy
environment {
  CI_ENVIRONMENT_URL = "${env.STAGING_URL}"
}
```

Important:

* must use **double quotes**
* otherwise Jenkins treats it as a literal string
* `${env.STAGING_URL}` gets resolved at runtime

## Pipeline execution result

After execution:

* `deploy staging` runs
* staging URL is extracted dynamically
* `staging e2e` runs tests against that URL
* reports show:

    * local E2E
    * staging E2E
    * production E2E

The pipeline now:

* deploys to staging
* automatically tests staging
* requires approval
* deploys to production
* tests production

Manual visual verification is no longer required.

## Important technical notes

### Quoting rules

* If a shell command contains single quotes,
* wrap the entire script string in double quotes

Example:

Correct:

```groovy
script: "node-jq -r '.deploy_url' deploy.output"
```

Incorrect:

```groovy
script: 'node-jq -r '.deploy_url' deploy.output'
```

### Why this solution works

The pipeline now:

1. Captures dynamic data
2. Stores it in an environment variable
3. Injects it into a later stage
4. Configures Playwright automatically

This removes the need for:

* hardcoded URLs
* manual testing
* guessing preview links
