# 52. Deploying to Production

## Objective of This Lecture

This lecture completes the CI/CD flow by **deploying the application to Netlify production using Jenkins**.

At this point, we already have everything required:

* ✅ Netlify CLI installed
* ✅ Netlify Site ID configured via environment variable
* ✅ Netlify authentication token securely injected from Jenkins credentials
* ✅ Build artifacts generated in the `build/` directory

Now we simply need to **execute the Netlify deploy command correctly**.

---

## Preconditions (What Must Already Exist)

Before deployment can work, the pipeline must already include:

1. **Authentication**

    * `NETLIFY_AUTH_TOKEN` provided via Jenkins credentials
2. **Target Site**

    * `NETLIFY_SITE_ID` environment variable
3. **Build Artifacts**

    * A `build/` directory containing the compiled website

If any of these are missing, deployment will fail.

---

## Netlify Deploy Command Overview

The Netlify CLI command we use is:

```
netlify deploy
```

However, by itself, this command is **not enough**. We must provide additional flags to clarify:

1. **What to deploy**
2. **Where to deploy**

---

## Required Deploy Flags

### 1. `--dir`

This flag tells Netlify **which directory to upload**.

In our project, the production-ready files live in:

```
build/
```

So we must specify:

```
--dir=build
```

Without this flag:

* Netlify does not know what to deploy
* Deployment will fail or do nothing

---

### 2. `--prod`

By default, Netlify deploys to a **preview environment**.

To deploy to **production**, we must explicitly add:

```
--prod
```

This ensures:

* The live website is updated
* The public Netlify URL changes immediately

---

## Final Deploy Command Used in Jenkins

Putting everything together:

```bash
netlify deploy --dir=build --prod
```

What this does:

* Authenticates using `NETLIFY_AUTH_TOKEN`
* Targets the site defined by `NETLIFY_SITE_ID`
* Uploads files from the `build/` directory
* Deploys directly to production

---

## Pipeline Execution Result

After committing the change and running the pipeline:

* Jenkins completes successfully
* The deploy stage finishes without errors
* Logs show:

    * Uploading files
    * Waiting for deploy
    * Deployment success confirmation

This indicates that **Netlify accepted the deployment**.

---

## Verifying the Deployment

Even if Jenkins reports success, **manual verification is still important** at this stage.

### How to Verify

1. Open the Netlify site URL (shown in logs or Netlify dashboard)
2. Refresh the page
3. Confirm:

    * The new application version is visible
    * The old manually uploaded page is gone

If the page reflects the latest build, deployment is confirmed.

---

## Why Manual Verification Is Still Needed (For Now)

At this stage:

* Jenkins confirms technical success
* Netlify confirms upload success

But **only a human check confirms correctness of content**

In upcoming lectures, this manual verification step will be **automated or reduced**.

## Key Takeaways

* `netlify deploy` is the core deployment command
* `--dir=build` specifies deployment artifacts
* `--prod` deploys to production (not preview)
* Jenkins + Netlify CLI now form a complete deployment pipeline
* Manual verification is still required (for now)
