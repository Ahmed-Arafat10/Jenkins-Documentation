# 47. Installing CLI Tools

## Goal of This Lecture

The goal of this lecture is to explain **how Jenkins can interact with Netlify without using a browser**, and to introduce the concept of **CLI (Command Line Interface) tools** as the correct way to automate external services.

Specifically, this lecture focuses on installing the **Netlify CLI** so Jenkins can deploy applications programmatically.

---

## Why Jenkins Does NOT Use a Browser

A very important concept is clarified at the beginning of this lecture:

* Jenkins **does not open websites**
* Jenkins **does not open browsers**
* Jenkins **does not click buttons**
* Jenkins **does not select files in a UI**

This kind of automation (UI automation) is:

* Slow
* Fragile
* Hard to maintain
* Not how DevOps automation works

Instead, **all automation is done via commands**.

---

## The Correct Approach: CLI Tools

To automate interactions with external platforms (like Netlify), we use **CLI tools**.

### What is a CLI?

A CLI (Command Line Interface):

* Is a program that runs entirely in the terminal
* Accepts commands and flags
* Can authenticate, upload files, trigger deployments, etc.
* Works perfectly with Jenkins and CI/CD pipelines

---

## Netlify CLI

Netlify provides an official CLI called:

**Netlify CLI**

Using this CLI, Jenkins will later be able to:

* Authenticate with Netlify
* Upload build artifacts
* Trigger deployments
* Manage sites

All without leaving the terminal.

---

## Prerequisites for Netlify CLI

* Netlify CLI is installed using **npm**
* npm comes with **Node.js**
* Therefore:

    * Jenkins must run this stage inside a **Node.js environment**
    * This is why **Docker with a Node image** is required

---

## Pipeline Preparation

### Reusing an Existing Structure

Instead of starting from scratch:

* An existing pipeline stage is copied
* This ensures:

    * Correct syntax
    * Correct indentation
    * Docker + Node.js already configured

A new stage is introduced:

```groovy
stage('deploy')
```

This stage will later handle deployment-related tasks.

---

## Installing Netlify CLI (First Attempt)

The command taken from Netlify documentation:

```bash
npm install -g netlify-cli
```

Then, to verify installation:

```bash
netlify --version
```

This is added inside the `deploy` stage and committed.

---

## Problem Encountered: Permission Denied

When the pipeline runs, it **fails** with a familiar error:

* `permission denied`
* Same issue previously seen when installing tools globally
* Root cause:

    * `-g` installs the tool **globally**
    * Global installs require elevated permissions
    * Docker container is **not running as root**

This is expected behavior.

---

## Important Lesson: Avoid Global Installs in CI

Using global installs (`npm install -g`) inside CI pipelines is generally **not recommended**, because:

* Requires root permissions
* Causes permission conflicts
* Can break workspace ownership
* Makes builds less reproducible

This is **not a Jenkins problem**, but a best-practice issue.

---

## Correct Solution: Local Installation

### Remove the `-g` Flag

Instead of installing globally:

```bash
npm install netlify-cli
```

This installs Netlify CLI:

* Locally inside the project
* Under `node_modules/`

---

## Running Netlify CLI Locally

Since the CLI is now installed locally, it must be executed using its path:

```bash
./node_modules/.bin/netlify --version
```

This ensures:

* No permission issues
* No need for root access
* Full compatibility with Jenkins and Docker

---

## Pipeline Result After Fix

After committing the fix and rerunning the pipeline:

* The **deploy stage succeeds**
* Netlify CLI installs successfully
* Version command outputs the installed Netlify CLI version
* Only warnings appear (expected with npm)
* No errors occur

This confirms:

* Netlify CLI is installed correctly
* Jenkins can execute it
* The environment is ready for automated deployment

---

## Key Observations from Logs

From the logs you should see:

* npm installing Netlify CLI locally
* Warning messages (safe to ignore)
* Output similar to:

```
Netlify CLI x.y.z
```

This confirms successful installation.

## Alternative Approach (Not Used Here)

Another valid approach (not used in this lecture) would be:

* Add `netlify-cli` to `package.json` dependencies
* Install it as part of `npm ci`

**AI: notes**

> This approach is also common and often preferred in real projects, because it locks the CLI version and improves reproducibility. The course intentionally shows an alternative method so you understand how to install tools that are *not already part of the project*.
