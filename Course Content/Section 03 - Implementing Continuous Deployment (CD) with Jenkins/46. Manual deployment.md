# 46. Manual Deployment

## Purpose of This Lecture

This lecture introduces **manual deployment** as a foundation for later automation.
Before automating deployments with Jenkins, it is critical to **fully understand the manual steps** involved in deploying an application.

---

## Deployment Platform Used

The deployment in this lecture is done using **Netlify** via its **manual deployment interface**.

Netlify allows you to deploy static web applications by uploading a pre-built directory that contains your website files.

---

## Prerequisites

* A **ZIP archive** provided with the lecture resources
* The ZIP archive must contain:

    * A `build/` directory
    * Inside it, at least one file: `index.html`

After extracting the ZIP file, you should see the `build` directory directly.

---

## Manual Deployment Steps

### 1. Access Manual Deployment

* From the Netlify dashboard, choose the option:
  **Deploy manually**

This opens an upload interface.

---

### 2. Upload the Build Directory

You have two options:

* **Drag and drop** the entire `build/` directory into the upload area
* Or click **Browse to upload** and select the `build` directory manually

Important:

* Upload the **directory itself**, not just the `index.html` file
* Netlify expects the directory structure, not a single file

---

### 3. Confirm Upload

* After selecting the directory, Netlify asks for confirmation
* Confirm that you trust the site
* Netlify begins uploading the files

During this phase:

* A message like *“Uploading, please don’t refresh”* is shown
* The process usually takes only a few seconds

---

### 4. Deployment Result

Once the upload finishes, Netlify shows:

* **Deploy success**

Netlify automatically:

* Processes the uploaded files
* Deploys the website
* Generates a **public URL**

---

### 5. Access the Deployed Application

* Netlify generates a **random site name**
* Click **Open Production Deploy**

You will see:

* The deployed website
* The contents of `index.html` rendered in the browser

Each deployment:

* Gets its **own unique URL**
* Uses a randomly generated name

---

## Site Name Configuration

* Netlify allows you to **change the site name**
* Not all names are available
* It is usually best to:

    * Use a random or unique name
    * Avoid relying on specific names unless required

This step is optional and not required for the course.

---

## Why This Matters for Automation

This lecture is intentionally manual.

To automate deployments later using Jenkins, you must first understand:

* What files are deployed
* Where those files come from
* How deployment success is verified
* What the expected final result looks like

### Manual Steps Identified (Important for Automation)

1. Obtain the `build` directory
2. Upload the directory to Netlify
3. Trigger deployment
4. Receive a deployment URL
5. Verify the deployed application in the browser

In upcoming lectures:

* Jenkins will perform **these exact steps automatically**
* No manual upload will be required
* Deployment verification will also be automated

---

## Key Takeaway

Manual deployment is not the goal — **understanding the process is**.
Once the steps are clear, automation becomes straightforward and reliable.
