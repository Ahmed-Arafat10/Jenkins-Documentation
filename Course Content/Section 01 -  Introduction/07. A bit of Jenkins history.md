# Lecture 7: A Bit of Jenkins History

## Why Jenkins Has Freestyle Jobs and Pipeline Jobs

### 1. Freestyle Jobs (Legacy Approach)

* **Origin**: Initially, Jenkins only supported **freestyle jobs**, simply called *jobs* at the time.
* **Configuration Method**:

    * Set up through the **Jenkins UI** (point-and-click configuration).
    * The configuration is stored **internally in Jenkins**, not as code.
* **Drawbacks**:

    * **No change tracking**: You can’t easily tell *who* changed a job, *when*, or *what* was changed.
    * **Debugging difficulties**: If a job breaks, it’s unclear whether the cause is a configuration change or an external factor.
    * **Limited flexibility**:

        * Execution flow is **linear** and lacks advanced control.
        * To extend functionality, many **plugins** must be installed and configured manually.

### 2. Pipeline Jobs (Modern Approach)

* **Pipeline as Code**:

    * Introduced as CI/CD needs grew more complex.
    * Configurations are stored as **code**, typically in a `Jenkinsfile`.
    * Enables **version control** (e.g., Git) for job configurations, just like application code.
* **Advantages**:

    * Full **auditability**: Track changes, authors, and history.
    * **Complex workflows**: Support for parallel execution, multiple stages, and conditional logic.
    * **Reduced reliance on plugins**: Many capabilities are natively supported.

### 3. Practical Usage Today

* **Freestyle Jobs**:

    * Still exist and are fine for **quick, simple tasks**.
    * Often found in **legacy systems** that have not migrated to pipelines.
    * Easy to set up but discouraged for new projects.
* **Pipeline Jobs**:

    * The **recommended standard** for modern DevOps practices.
    * Align with Infrastructure as Code (IaC) and CI/CD best practices.
    * Provide **scalability, maintainability, and collaboration benefits**.

---

## Key Takeaway

* **Freestyle jobs** were the original Jenkins jobs: simple but limited, stored in the UI without code.
* **Pipeline jobs** are the future and present of Jenkins: configuration as code, better change tracking, and support for complex CI/CD pipelines.
* In this course, we will focus only on **pipeline jobs** to follow the most up-to-date practices.
