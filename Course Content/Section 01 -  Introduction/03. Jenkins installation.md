# Lecture 3: Jenkins Installation

## Overview

This lecture explains how to install Jenkins using Docker Desktop to ensure a consistent environment for the course. The lecturer recommends Docker-based installation to avoid compatibility issues across different systems.

---

## Step-by-Step Installation

### 1. Install Docker Desktop

* Download and install **Docker Desktop** for your operating system (Windows, macOS, or Linux).
* Create a **Docker Hub account** (recommended, though optional).
* Resources provided in the course folder include:

    * Detailed installation instructions
    * Troubleshooting tips

### 2. Verify Docker Installation

Run the following command in a terminal to confirm Docker is working:

```bash
docker run hello-world
```

* If you see `Hello from Docker!`, Docker is correctly installed.
* A small "hello-world" container will appear in Docker Desktop. This can be removed afterward.

### 3. Get Jenkins Docker Setup Repository

* Clone the prepared repository (or download the ZIP):

```bash
git clone <repo-url>
cd install-jenkins-docker
```

### 4. Build Jenkins Docker Image

Inside the cloned directory:

```bash
docker build -t my-jenkins .
```

* This creates a Docker image containing Jenkins.
* Only needs to be run **once**.

### 5. Start Jenkins with Docker Compose

Run:

```bash
docker compose up -d
```

* First run takes time as images are downloaded.
* After the first run, startup is much faster.
* If no errors appear and containers are green, Jenkins is running.

### 6. Access Jenkins

* Open browser and go to:

  ```
  http://localhost:8080
  ```
* Jenkins UI will appear but needs initial setup.

## Jenkins Initial Setup

### 1. Unlock Jenkins

* Jenkins asks for an **initial admin password**.
* To retrieve it, either:

    * Check **container logs** in Docker Desktop
    * Or use `exec` to run inside container:

      ```bash
      docker exec -it <jenkins-container-name> cat /var/jenkins_home/secrets/initialAdminPassword
      ```
* Copy the password, paste into Jenkins setup, and proceed.

### 2. Install Suggested Plugins

* Select **“Install Suggested Plugins”**.
* Jenkins installs commonly used plugins (takes a few minutes).

### 3. Create Admin User

* Create a new user with:

    * Username
    * Secure password
    * Email (example: `valentine@example.com`)
* **Do not** rely on the initial admin password long term.

### 4. Jenkins URL Setup

* Jenkins automatically sets URL to:

  ```
  http://localhost:8080
  ```
* This is mapped from the Docker container → local machine.
* Bookmark it for future use.

### 5. Finish Setup

* Click **Save & Finish → Start using Jenkins**.
* Jenkins is now ready to use.

## Managing Jenkins with Docker

### Stop Jenkins

From inside the `install-jenkins-docker` directory:

```bash
docker compose down
```

* Stops all Jenkins containers.
* Jenkins becomes unavailable on `localhost:8080`.

### Pause/Stop Docker Desktop

* You can **pause** Docker Desktop to save resources.
* Or completely **quit** Docker Desktop if not needed.

### Restart Jenkins

1. Start Docker Desktop.
2. Inside the repo directory, run:

   ```bash
   docker compose up -d
   ```
3. Access Jenkins again at `http://localhost:8080`.
4. Log in with the admin user you created.

---

## Key Takeaways

* Use **Docker Desktop** for a clean, reproducible Jenkins setup.
* Verify installation with `docker run hello-world`.
* Clone the prepared **Jenkins Docker repository**.
* Build once, then start Jenkins using `docker compose up`.
* Unlock Jenkins with the **initial admin password** from container logs.
* Install **suggested plugins** and create a new **admin user**.
* Access Jenkins at `http://localhost:8080`.
* Manage Jenkins with `docker compose down` (stop) and `docker compose up` (restart).

## AI: Notes

* 📌 Instead of `docker compose up`, you can run it in detached mode with:

  ```bash
  docker compose up -d
  ```

  This keeps Jenkins running in the background without blocking your terminal.

* 📌 If you need to see logs later, use:

  ```bash
  docker compose logs -f
  ```

* 📌 To avoid typing the full container name each time, you can list running containers:

  ```bash
  docker ps
  ```

* 📌 Always keep Docker Desktop updated, as older versions can cause compatibility issues.

- Once you are done playing with Jenkins, maybe it is time to clean things up.
- Run the following command to terminate Jenkins and to remove all volumes and images used:
  ```bash
  docker compose down --volumes --rmi all
  ```
