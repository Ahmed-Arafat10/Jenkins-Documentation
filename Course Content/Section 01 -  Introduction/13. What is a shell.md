# Lecture 13 – What is a Shell?

## Introduction

In this lecture, we explore what a **shell** is, why it matters, and how it connects to the commands we use in Linux and Jenkins automation. Understanding the shell is crucial since Jenkins pipelines often execute Linux commands in sequence.

---

## Key Concepts

### 1. Command Line Interface (CLI) vs Graphical User Interface (GUI)

* **GUI (Graphical User Interface):**

    * Example: Right-click → *Create Folder* → open a text editor → add a file.
    * Designed for human interaction.

* **CLI (Command Line Interface):**

    * Example: Using commands like `mkdir`, `touch`, `echo`, `cat`.
    * No graphical interaction—everything is text-based.
    * More reliable and automation-friendly than GUI automation.

### 2. What is a Shell?

* A **shell** is the program that provides the **CLI interface** to the operating system.

* It is the **outer layer** of the system—the part we see and interact with.

* Functions of the shell:

    1. Accepts commands from the user.
    2. Passes them to the operating system for execution.
    3. Returns the results/output.

* Common shells:

    * `sh` (Bourne Shell)
    * `bash` (Bourne Again Shell)
    * `zsh`, `fish`, etc.

> In Jenkins, when we run a job, we are essentially asking the shell to execute a sequence of commands.

### 3. Why is the Shell Important in Jenkins?

* Jenkins **automates shell commands**.
* Pipelines run sequences of Linux commands in a strict order.
* Example:

  ```bash
  mkdir new_project   # Create a directory
  touch new_project/file.txt  # Create a file inside it
  ```

    * If order is incorrect (e.g., creating file before directory), commands will fail.

### 4. Practical Takeaway

* You should **practice Linux commands** on your own system to gain confidence.
* The course will introduce more commands progressively.
* Resources are available in the provided **resources folder** for setting up a Linux environment locally.

---

## Summary

* **Shell** = interface that lets us interact with the system using text commands.
* **CLI vs GUI** = CLI is better for automation (Jenkins relies on it).
* **Jenkins pipelines** = structured sequences of shell commands.
* **Order of execution matters** (e.g., create directory → then add file).
* **Hands-on practice** is essential for mastering shell usage.
