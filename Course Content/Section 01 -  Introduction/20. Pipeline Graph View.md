# 20. Pipeline Graph View

## Overview

Jenkins provides multiple ways to visualize pipeline executions. Apart from the **Stage View** (the classic tabular representation), there is a newer and more interactive view called the **Pipeline Graph View**.

The Pipeline Graph View offers a graphical representation of stages, actions, and results, making it easier to drill down into pipeline details.

---

## Accessing the Pipeline Graph View

* Navigate to the pipeline’s **status page**.
* From the menu, select **Stages**.
* This will open the **Pipeline Graph View** interface.

---

## Features of Pipeline Graph View

1. **Entire Pipeline Overview**

    * Displays all stages in a visual sequence.
    * Each stage is represented as a block/node in the graph.

2. **Stage-Level Details**

    * Clicking on a stage expands its details.
    * You can see:

        * Messages that were printed.
        * Shell scripts that were executed.
        * Outputs and results for that stage.

3. **Expandable Sections**

    * Each action within a stage (e.g., `echo`, `sh`, or test execution) can be expanded to inspect logs and results.

4. **Consistent with Other Views**

    * Works similarly to the Stage View but provides a more modern and interactive interface.

---

## Additional Options in Graph View

From within the Pipeline Graph View, you can also:

* **Start a build** directly.
* **View the console output** of the pipeline run.
* **Configure the job** from the same interface.

---

## Notes & Relevance

* The **Pipeline Graph View** is a **newer Jenkins feature** compared to the classic Stage View.
* The course does not rely on this view, but it is useful to be aware of its existence for exploration and troubleshooting.

---

## Key Takeaways

1. The Pipeline Graph View is an **alternative visualization** of pipeline stages.
2. It provides a **graphical and interactive** way to inspect pipelines.
3. It allows **expanding stages** to review commands, outputs, and logs.
4. You can also perform actions like starting builds, viewing console logs, and configuring jobs from this view.

---

## (AI: notes)

* The Graph View is part of Jenkins’ effort to modernize the UI. Depending on the Jenkins version, this may require the **Pipeline Graph View Plugin**.
* Some organizations prefer this view for CI/CD observability since it gives a **cleaner timeline of stage execution** compared to the classic Stage View.
* For long pipelines, this view makes navigation easier since it reduces clutter compared to tabular representation.
