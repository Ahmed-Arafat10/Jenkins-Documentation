## 21. What is DevOps?

### ❌ What DevOps is NOT

* Not a **standard** or **specification**.
* Not a **tool** or single piece of software.
* Not simply "using a particular set of tools".

### ✅ What DevOps IS

* A **cultural shift** and **mindset change**.
* A set of **practices** aiming to improve collaboration between different roles.
* Closely tied to the **Agile movement** (continuous iterations, adapting to change).

---

### 🔹 Traditional Workflow Problem

1. Customer requests feature → Project Manager writes specs.
2. Developers build → Testers test → Sysadmins deploy.
3. Many "handovers" and **silos** → leads to miscommunication & blame.
4. Developers know little about infrastructure.
5. Ops team knows little about the software.

---

### 🔹 DevOps Approach

* **Collaboration over silos** → all stakeholders (PMs, Devs, Testers, Ops) work together.
* Everyone takes **responsibility for the final outcome**.
* **Blame culture** replaced with **learning and experimentation**.
* Final product success = **everyone wins**.

---

### 🔹 Automation in DevOps

* Manual, repetitive work = **productivity killer**.
* DevOps focuses on:

    * **Automation** of builds & deployments → **CI/CD**.
    * Using **CLI tools**, **shell scripting**, **documentation**.

---

### 🔹 DevOps Lifecycle (Infinity Loop)

1. **Plan** → **Code** → **Build** → **Test** → **Release** → **Deploy** → **Operate** → **Monitor** → feedback → repeat.

* Continuous loop = continuous improvement.
* Feedback incorporated into every iteration.

---

### 🔹 DevOps & Agile

* DevOps complements **Agile & Scrum**.
* Both emphasize:

    * Iterative development.
    * Quick feedback.
    * Adapting to changes.
* Knowing Agile/Scrum = highly valuable in modern organizations.

---

### 📚 Recommended Resource

* **The Phoenix Project** (a novel about DevOps culture, transition & challenges).

---

### 📝 Key Takeaways

* DevOps = **culture + practices + automation**, not just tools.
* Goal = **better collaboration, faster delivery, higher quality software**.
* Tools support DevOps, but adopting tools alone ≠ doing DevOps.
