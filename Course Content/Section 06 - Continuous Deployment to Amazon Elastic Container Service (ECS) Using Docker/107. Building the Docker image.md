## Lecture 107 — Building the Docker Image

**Section:** Continuous Deployment to Amazon Elastic Container Service (ECS) Using Docker

---

## Goal of This Step

Now that a **Dockerfile** exists in the project, the next step is to **build the Docker image inside the Jenkins
pipeline**.

The Docker image will contain:

* the web server
* the application files
* all runtime dependencies

This image will later be pushed to:

Amazon Elastic Container Registry

and deployed to:

Amazon Elastic Container Service

---

# Adjusting the Jenkins Pipeline Order

The pipeline must first **build the application** and then **build the Docker image**.

Therefore, the deployment stage must appear **after the build stage**.

Correct stage order:

```id="n5o0rt"
Build Project
      ↓
Build Docker Image
      ↓
Deploy to AWS
```

The **Deploy to AWS** stage is moved to the end of the pipeline.

---

# Creating the Docker Build Stage

A new pipeline stage is created:

```groovy id="q0p3c3"
stage('Build Docker Image') {
    steps {
        sh 'docker build -t my-jenkins-app .'
    }
}
```

### Explanation

| Part                | Purpose                     |
|---------------------|-----------------------------|
| `docker build`      | Builds a Docker image       |
| `-t my-jenkins-app` | Assigns a tag to the image  |
| `.`                 | Specifies the build context |

---

# Understanding the Build Context

The dot (`.`) at the end of the command is very important.

```bash id="od6d1e"
docker build -t my-jenkins-app .
```

The dot represents the **build context directory**.

This means Docker will use the **current project directory** as the source for files referenced in the Dockerfile.

For example, if the Dockerfile contains:

```dockerfile id="z3pxr3"
COPY build /usr/share/nginx/html
```

Docker must have access to the `build` directory.

If the build context is incorrect, Docker will fail to locate these files.

After committing the changes and running the pipeline:

The **Build Docker Image stage completes successfully**.

Typical output shows two steps:

### Step 1 — Pull Base Image

Docker downloads the base image defined in the Dockerfile.

Example:

```id="x5knkj"

```

### Step 2 — Copy Application Files

Docker copies the website files into the container image.

Example:

```id="zvafos"

```

After these steps, the Docker image is successfully created.

# Preparing for the Next Step

Although the image has been built successfully, the pipeline will later need to:

1. **push the image to Amazon ECR**
2. **update the ECS deployment**

For that reason, the AWS deployment stage requires access to the **AWS CLI container**.

---

# Running AWS CLI in the Pipeline

The pipeline uses an AWS CLI container image.

Example configuration:

```groovy id="by66n2"
agent {
    docker {
        image 'amazon/aws-cli'
        args '-u root'
        reuseNode true
    }
}
```

Important parameters:

| Option      | Purpose                    |
|-------------|----------------------------|
| `image`     | AWS CLI container          |
| `-u root`   | Run container as root user |
| `reuseNode` | Reuse Jenkins workspace    |

---

# New Problem: Docker Command Not Found

When the pipeline runs again, it fails with:

```id="p5eixb"
docker: command not found
```

This happens because the **AWS CLI container does not include Docker**.

The container contains:

* AWS CLI tools
* basic Linux utilities

But it does not contain the Docker client.

---

# Installing Docker in the AWS CLI Container

The container is based on **Amazon Linux**, which uses the `yum` package manager.

To install Docker, the pipeline runs:

```bash id="9apwhl"
yum install docker -y #Wrong: amazon-linux-extras install docker
```

Explanation:

| Command               | Purpose                      |
|-----------------------|------------------------------|
| `amazon-linux-extras` | Amazon Linux package manager |
| `install docker`      | Installs Docker client       |

(Added explanation – not mentioned directly in the lecture)
`amazon-linux-extras` is used to install optional packages in Amazon Linux distributions.

---

# Second Error: Cannot Connect to Docker Daemon

After installing Docker, the pipeline fails again with:

```id="sv6uxn"
Cannot connect to the Docker daemon
```

This error occurs because Docker has **two main components**.

### Docker Client

The command-line tool:

```bash id="rtjyt1"
docker
```

### Docker Daemon

The background service responsible for:

* building images
* running containers
* managing Docker resources

The pipeline container now has the **Docker client**, but the **Docker daemon is running on the Jenkins host**.

Therefore the client cannot communicate with it.

---

# Understanding the Docker Socket

Docker uses a Unix socket file for communication:

```id="d2g33g"
/var/run/docker.sock
```

This file enables communication between:

* Docker client
* Docker daemon

To allow the container to access the host daemon, the socket must be **mounted into the container**.

---

# Mapping the Docker Socket

Docker socket mapping is added to the container configuration.

Example:

```groovy id="kz92o4"
args '-u root -v /var/run/docker.sock:/var/run/docker.sock'
```

Explanation:

| Part                   | Purpose                      |
|------------------------|------------------------------|
| `-v`                   | Mount volume                 |
| `/var/run/docker.sock` | Docker socket on host        |
| `/var/run/docker.sock` | Socket path inside container |

This creates a connection between the **container Docker client** and the **host Docker daemon**.

---

# Why This Works

The architecture now looks like this:

```id="zt98uf"
Jenkins Container
   │
   │ Docker Client
   ▼
Docker Socket (/var/run/docker.sock)
   │
   ▼
Docker Daemon (Host Machine)
   │
   ▼
Build Docker Image
```

This allows the pipeline container to **control Docker running on the Jenkins host**.

### Jenkinsfile so far

````
pipeline {
    agent any
    environment {
        AWS_DEFAULT_REGION = 'us-east-1'
        AWS_ECS_CLUSTER = 'LearnJenkinsAppClusterProd'
        AWS_ECS_SERVICE_PROD = 'learn-jenkins-app-task-definition-prod-service-vd0czv2v'
        AWS_ECS_TD_PROD = 'learn-jenkins-app-task-definition-prod'
    }
    stages {
        stage('Build App') {
            agent {
                docker {
                    image 'node:18-alpine'
                    reuseNode true
                }
            }
            steps {
                sh '''
                echo "=== Workspace before build ==="
                ls  -lah

                echo "=== Node & NPM versions ==="
                node -v
                npm -v

                echo "=== Installing dependencies ==="
                npm ci

                echo "=== Running build ==="
                npm run build

                echo "=== Workspace after build ==="
                ls -lah
                '''
            }
        }
        stage('Build Docker Image') {
            agent {
                docker {
                    image 'amazon/aws-cli'
                    reuseNode true
                    args "-u root -v /var/run/docker.sock:/var/run/docker.sock --entrypoint=''"
                }
            }
            steps {
                sh '''
                yum install docker -y
                docker build -t custom-nginx .
                '''
            }
        }
        stage('Deploy to AWS ECS') {
            agent {
                docker {
                    image 'amazon/aws-cli'
                    reuseNode true
                    args "-u root --entrypoint ''"
                }
            }
            steps {
                withCredentials([usernamePassword(credentialsId: 'aws-ecs', passwordVariable: 'AWS_SECRET_ACCESS_KEY', usernameVariable: 'AWS_ACCESS_KEY_ID')]) {
                    sh '''
                    yum install -y jq
                    aws --version
                    LATEST_TD_REVISION=$(aws ecs register-task-definition --cli-input-json file://aws/task-definition-prod.json | jq '.taskDefinition.revision')

                    aws ecs update-service \
                     --service $AWS_ECS_SERVICE_PROD \
                     --task-definition $AWS_ECS_TD_PROD:$LATEST_TD_REVISION \
                     --cluster $AWS_ECS_CLUSTER

                    aws ecs wait services-stable \
                     --services $AWS_ECS_SERVICE_PROD \
                     --cluster $AWS_ECS_CLUSTER

                    echo "Deployment to AWS ECS completed successfully."
                    '''
                }
            }
        }
    }
}
```
