## ✅ Steps for Using a Git Repository in Jenkins

### 1. Create a Jenkinsfile in Your Repo

* Inside your project root (not inside a subfolder).
* File name must be **exactly** `Jenkinsfile` (case-sensitive).
* Define a simple pipeline there (even the sample one Jenkins provides is fine).

Example minimal Jenkinsfile:

```groovy
pipeline {
    agent any
    stages {
        stage('Hello') {
            steps {
                echo 'Hello Jenkins!'
            }
        }
    }
}
```

---

### 2. Commit & Push the Jenkinsfile to GitHub

* Use `git add .`
* `git commit -m "Add Jenkinsfile"`
* `git push origin main`
* Make sure the file is visible in your GitHub repository.

---

### 3. Configure Jenkins to Use the Repository

1. In Jenkins, create a **New Item** → give it a name (e.g., *Learn Jenkins App*).
2. Choose **Pipeline** → click **OK**.
3. Under **Pipeline** settings:

    * Set **Definition** = *Pipeline script from SCM*.
    * Set **SCM** = *Git*.
    * Enter your **Repository URL** (HTTPS or SSH).
    * If private, provide **credentials** (username/password or SSH key).
    * Set **Branch Specifier** = `main` (instead of default `master`).
    * Leave **Script Path** = `Jenkinsfile` (default, since we put it in root).

---

### 4. Run the Pipeline

* Click **Save** → **Build Now**.
* Jenkins will:

    * Clone your GitHub repo.
    * Locate the `Jenkinsfile`.
    * Execute the pipeline stages defined there.

---

### 5. Making Changes Later

* Edit your `Jenkinsfile` in GitHub (locally or Codespaces).
* `git commit` + `git push`.
* Go to Jenkins → click **Build Now** again to test changes.
* (Later you’ll learn to set up **webhooks** so Jenkins builds automatically on push.)

---

## 🔑 Key Points to Remember

* `Jenkinsfile` must be in repo **root**.
* File name must be `Jenkinsfile` (no extensions).
* Always push changes before running in Jenkins.
* Default branch is `main`, not `master`.
* For private repos, Jenkins needs credentials.
