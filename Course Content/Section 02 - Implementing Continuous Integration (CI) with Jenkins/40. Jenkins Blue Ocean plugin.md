## 🔹 The Problem

* Default Jenkins **Stage View** is limited.
* When jobs run in **parallel**, the classic view doesn’t clearly show the pipeline flow.
* Harder to visualize complex pipelines.

---

## 🔹 The Solution → Blue Ocean

* **Blue Ocean** is a Jenkins plugin (actually a collection of plugins) that provides a **modern UI**.
* It doesn’t replace Jenkins functionality, only **improves visualization**.
* Jobs, pipelines, and configs still work exactly as before.

---

## 🔹 Installing Blue Ocean

1. Go to **Dashboard → Manage Jenkins → Plugins → Available Plugins**.
2. Search for **Blue Ocean**.

    * Install **Blue Ocean Aggregator** (it automatically pulls in all required sub-plugins).
    * Make a note/screenshot of which plugins get installed in case you want to uninstall later.
3. After install, check **“Restart Jenkins when installation is complete and no jobs are running”**.
4. Log back in → Jenkins dashboard will now show a new option: **Open Blue Ocean**.

---

## 🔹 Features of Blue Ocean

* **Pipeline visualization**:

    * Shows stages as a flow diagram.
    * Clearly represents **parallel stages** (unit test + e2e test side by side).
* **Test results tab**: view which tests ran and their status.
* **Artifacts tab**: view/download published build artifacts.
* **Logs tab**: cleaner log viewer compared to classic Jenkins.
* **Run & replay builds** easily.
* Can switch back anytime to **Classic view**.

---

## 🔹 Example Visualization

* Classic view → sequential-looking stages (parallel stages hard to spot).
* Blue Ocean view → proper diagram:

  ```
  Checkout → Build → [Unit Tests || E2E Tests] → Deploy → Finish
  ```
* Much clearer for debugging pipeline setup.

---

## 🔹 When to Use

* Especially useful when:

    * You have **parallel stages** (tests, builds, deploys).
    * You want **better visualization** of pipeline flow.
* Some long-time Jenkins users stick to Classic view, but Blue Ocean is very helpful for new users and complex pipelines.

---

✅ **In short**:
Blue Ocean makes Jenkins pipelines **easier to read, debug, and present** — especially when using **parallel stages**.
