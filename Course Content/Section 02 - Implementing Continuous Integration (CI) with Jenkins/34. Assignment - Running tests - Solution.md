### Lecture 34 — Assignment: Running Tests (Solution) ✅

In this lecture, we extend our Jenkins pipeline to **add a test stage** and run actual application tests inside a Docker container.

---

#### 1. Adding a New Stage

* We place a new `stage('Test')` **after the build stage**.
* At first, we only echo `"test stage"` to validate syntax and stage creation.
* After committing & running the pipeline, Jenkins shows the new stage and its logs.

---

#### 2. Checking Build Artifacts

* Replace the dummy `echo` with a file existence check:

  ```sh
  test -f build/index.html
  ```
* This ensures the **build artifact exists**.
* If the file is missing, the stage fails.

---

#### 3. Running Application Tests

* The application has:

    * `app.js` → source code
    * `app.test.js` → very simple **unit test**
* To run tests, we use:

  ```sh
  npm test
  ```
* But `npm test` requires a **Node.js runtime** → we use **Docker agent with Node.js image** inside the test stage.

---

#### 4. Final Test Stage Code (conceptual)

```groovy
stage('Test') {
    agent {
        docker { image 'node:18-alpine' }
    }
    steps {
        sh '''
            test -f build/index.html
            npm test
        '''
    }
}
```

---

#### 5. Pipeline Behavior

* Jenkins spins up a **Node.js container**.
* Runs the file check → confirms build artifact exists.
* Runs `npm test` → executes the unit test successfully.
* Logs show: **1 passed, 0 failed**.

---

#### 6. Dependency Between Stages

* Current order: **Build → Test**.
* Why? Because the **Build stage installs dependencies (`npm ci`)** into `node_modules`.
* If we swap them, `npm test` fails unless we also add `npm ci` to the test stage.
* This shows the importance of **stage dependencies**.

---

✅ **Key Takeaways**

* Pipelines should evolve step by step (add, test, refine).
* Always validate existence of **build artifacts**.
* Use **Docker agent** to bring runtime environments (Node.js in this case).
* Understand **dependencies between stages** → order matters unless you re-install dependencies.
