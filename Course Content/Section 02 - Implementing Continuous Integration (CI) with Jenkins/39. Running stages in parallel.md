## 🔹 The Problem

* By default, Jenkins executes **stages sequentially** (one after another).
* Example: `checkout → build → unit test → e2e test → reports`.
* This can waste time, especially when some tests don’t depend on each other.

---

## 🔹 The Solution → `parallel` block

* Jenkins allows you to run **multiple stages in parallel** inside a parent stage.
* You define a **wrapper stage** and use the `parallel` keyword inside it.

---

### ✅ Example: Running Unit Tests & End-to-End Tests in Parallel

```groovy
pipeline {
    agent any

    stages {
        stage('Checkout') {
            steps {
                checkout scm
            }
        }

        stage('Run Tests') {
            parallel {
                stage('Unit Test') {
                    steps {
                        echo "Running unit tests..."
                        sh 'npm run test:unit'
                    }
                    post {
                        always {
                            junit 'reports/junit/*.xml'
                        }
                    }
                }

                stage('End-to-End Test') {
                    steps {
                        echo "Running end-to-end tests..."
                        sh 'npx playwright test --reporter=html'
                    }
                    post {
                        always {
                            publishHTML([
                                reportDir: 'playwright-report',
                                reportFiles: 'index.html',
                                reportName: 'Playwright Report'
                            ])
                        }
                    }
                }
            }
        }
    }
}
```

---

## 🔹 Key Notes

* `stage('Run Tests')` → wrapper stage.
* `parallel { ... }` → tells Jenkins to execute inner stages **at the same time**.
* Inner stages (`Unit Test`, `End-to-End Test`) each have their own **steps** and **post actions** (e.g., JUnit for unit tests, HTML publisher for Playwright).
* Logs may look **interleaved** since both stages write simultaneously, but they **really run in parallel**.

---

## 🔹 Benefits

* Saves CI/CD pipeline runtime (tests run simultaneously).
* Keeps reports separated and tied to their stage.
* Cleaner structure since each stage can have its own `post` block.

---

👉 In short:
Wrap stages inside a parent `stage { parallel { ... } }`, and Jenkins will execute them in **parallel**, while still letting you keep **per-stage reports** and post actions.


- updated Jenkinsfile
````
pipeline {
    agent any

    stages {
        /*

        stage('Build') {
            agent {
                docker {
                    image 'node:18-alpine'
                    reuseNode true
                }
            }
            steps {
                sh '''
                    ls -la
                    node --version
                    npm --version
                    npm ci
                    npm run build
                    ls -la
                '''
            }
        }
        */

        stage('Tests') {
            parallel {
                stage('Unit tests') {
                    agent {
                        docker {
                            image 'node:18-alpine'
                            reuseNode true
                        }
                    }

                    steps {
                        sh '''
                            #test -f build/index.html
                            npm test
                        '''
                    }
                    post {
                        always {
                            junit 'jest-results/junit.xml'
                        }
                    }
                }

                stage('E2E') {
                    agent {
                        docker {
                            image 'mcr.microsoft.com/playwright:v1.39.0-jammy'
                            reuseNode true
                        }
                    }

                    steps {
                        sh '''
                            npm install serve
                            node_modules/.bin/serve -s build &
                            sleep 10
                            npx playwright test  --reporter=html
                        '''
                    }

                    post {
                        always {
                            publishHTML([allowMissing: false, alwaysLinkToLastBuild: false, keepAll: false, reportDir: 'playwright-report', reportFiles: 'index.html', reportName: 'Playwright HTML Report', reportTitles: '', useWrapperFileDirectly: true])
                        }
                    }
                }
            }
        }
    }
}
````
