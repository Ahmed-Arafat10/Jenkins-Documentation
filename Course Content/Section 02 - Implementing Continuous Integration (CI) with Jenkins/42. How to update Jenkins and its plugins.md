## 🔹 Why Updates Matter

* Warnings may appear in Jenkins:

    * New Jenkins core version available.
    * Security issues in plugins.
* Updates are mainly **security-related** (core + plugins).

---

## 🔹 Best Practices Before Updating

1. **Communicate** → Discuss with team/impacted parties.
2. **Plan downtime** → Pick a time of minimal impact.
3. **Document** → Keep a log of:

    * Date.
    * Component (Core / Plugin).
    * Reason (e.g., Security).
    * Old version.
    * New version.
    * Who updated.
    * Status.
4. **Update gradually** → One update at a time for easier troubleshooting.

---

## 🔹 Updating Jenkins Core (Docker Setup)

1. **Check current version**

    * Bottom of Jenkins dashboard.
    * Record in spreadsheet.

2. **Stop Jenkins**

   ```bash
   docker compose down
   ```

3. **Rebuild image (to pull latest Jenkins base image)**

   ```bash
   docker build -t my-jenkins .
   ```

4. **Restart Jenkins**

   ```bash
   docker compose up -d
   ```

5. **Verify**

    * Log back in → Check version at bottom of page.
    * Update spreadsheet (old vs new version).

---

## 🔹 Updating Plugins

1. **Document first**

    * Example: GitHub plugin (security update).
    * Note old version + update reason.

2. **Go to Plugin Manager → Updates tab.**

    * Search plugin (e.g., `GitHub`).
    * Compare **Installed version** vs **Available update**.

3. **Update plugin**

    * Select plugin → Click **Update**.
    * Download & prepare for activation.

4. **Restart Jenkins** (two options):

    * Via GUI (after updates complete).
    * Or special URL:

      ```
      http://<jenkins-url>:8080/restart
      ```

5. **Verify**

    * Log back in → No warnings.
    * Check plugin under **Installed plugins** → Confirm new version.
    * Update spreadsheet (success).

---

## 🔹 Extra Tips

* **Do light testing** after updates (jobs still run as expected).
* **Stagger plugin updates** → 1 per day (if something breaks, easier to identify).
* **Use internal wiki/docs** → Keep update history accessible to team.
* **Production Jenkins** → Apply security updates promptly.

---

✅ With these steps, you can safely keep **Jenkins core & plugins** up-to-date while minimizing downtime and troubleshooting effort.
